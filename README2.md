## Для разработчика

### Техническая архитектура

Проект реализован как клиент-серверное веб-приложение  
Frontend написан на HTML, CSS и JavaScript 
Backend реализован на Python с использованием Flask  
Для хранения данных используется SQLite

Пользователь взаимодействует с приложением через браузер
Frontend отвечает за отображение интерфейса, а backend — за игровую логику, комнаты, авторизацию и хранение состояния игры

Обмен данными между frontend и backend происходит через REST API в формате JSON

Для описания архитектуры используется модель C4:
- Context
- Container
- Component
- Code


# C1. Context diagram

На первом уровне система рассматривается как единое приложение

Пользователь открывает игру в браузере, проходит регистрацию или вход, создаёт комнату или подключается к уже существующей игре. После этого игрок попадает на игровой экран, где может покупать и продавать акции, использовать инсайд и завершать игровые дни

Веб-приложение обрабатывает действия пользователя, обновляет состояние игры и сохраняет данные в SQLite

База данных используется для хранения:
- пользователей
- комнат
- игроков
- баланса
- риска проверки
- портфелей
- истории капитала
- игровых событий

Основные действия пользователя:
- регистрация
- вход в аккаунт
- создание комнаты
- вход в публичную или приватную комнату
- покупка акций
- продажа акций
- покупка инсайда
- завершение дня
- просмотр итогов игры

## Frontend

Frontend отвечает за:

- отображение результатов игры

Основные frontend-файлы:

| Файл | Назначение |
|---|---|
| `welcome.html` | стартовая страница |
| `rules.html` | правила игры |

## Backend

Backend отвечает за:
- обработку HTTP-запросов
- авторизацию
- создание комнат
- подключение игроков
- игровую логику
- пересчёт состояния игры
- работу с базой данных

Основные backend-файлы:

| Файл | Назначение |
|---|---|
| `app.py` | запуск Flask-приложения |
| `auth_api.py` | авторизация |
| `rooms_api.py` | комнаты |
| `game_api.py` | игровые действия |
| `game_engine.py` | игровая логика |
| `database.py` | работа с SQLite |
| `config.py` | игровые константы |

## `app.py`

`app.py` является точкой входа в приложение

Он:
- создаёт Flask-приложение
- подключает API-модули
- раздаёт frontend-файлы
- запускает backend-сервер


## `auth_api.py`

Модуль отвечает за:
- регистрацию
- вход
- выход
- получение текущего пользователя

После успешного входа id пользователя сохраняется в `session`

## `rooms_api.py`

Модуль отвечает за:
- создание комнат
- список комнат
- вход в комнату
- приватные коды
- запуск игры

Комната может быть:
- публичной
- приватной

## `game_api.py`

Модуль отвечает за:
- получение состояния игры
- покупку акций
- продажу акций
- покупку инсайда
- завершение дня
- подтверждение игровых событий

## `game_engine.py`

Основной модуль игровой логики

Он отвечает за:
- построение состояния игры
- обновление цен
- пересчёт капитала
- рейтинг игроков
- завершение дня
- завершение игры
- игровые события


## `database.py`

Модуль отвечает за:
- подключение к SQLite
- создание таблиц
- выполнение SQL-запросов

## `config.py`

Файл хранит:
- игровые константы
- стартовый капитал
- стоимость инсайда
- длительность хода
- риск проверки
- стартовые данные компаний

# REST API

| Метод | URL | Назначение |
|---|---|---|
| `POST` | `/api/register` | регистрация |
| `POST` | `/api/login` | вход |
| `POST` | `/api/logout` | выход |
| `GET` | `/api/me` | текущий пользователь |
| `GET` | `/api/rooms` | список комнат |
| `POST` | `/api/rooms` | создание комнаты |
| `POST` | `/api/rooms/<id>/join` | вход в комнату |
| `POST` | `/api/rooms/join-by-code` | вход по коду |
| `GET` | `/api/rooms/<id>/state` | состояние игры |
| `POST` | `/api/rooms/<id>/trade` | покупка или продажа |
| `POST` | `/api/rooms/<id>/insider` | покупка инсайда |
| `POST` | `/api/rooms/<id>/finish-day` | завершение дня |
| `POST` | `/api/rooms/<id>/events/ack` | подтверждение события |
