# Для пользователя

## Инсайдер

«Инсайдер» — это многопользовательская экономическая игра, в которой игроки соревнуются друг с другом на собственном мини-фондовом рынке

Вам предстоит:

- покупать и продавать акции
- анализировать новости
- отличать правдивые слухи от ложных
- использовать инсайдерскую информацию
- рисковать ради прибыли
- избегать штрафов
- и пытаться заработать больше остальных игроков

Проект вдохновлён механиками реального фондового рынка, но при этом остаётся понятным даже людям без знаний экономики


# Почему проект интересный

В отличие от обычных симуляторов биржи, здесь рынок действительно живой

Цена акций меняется не только из-за новостей, но и из-за действий игроков

Если все начинают массово покупать акции одной компании:

- цена начинает расти
- формируется пузырь
- а потом рынок может резко обвалиться

Из-за этого каждая партия развивается по-разному

Иногда выгодно играть осторожно. Иногда — идти ва-банк и рисковать ради огромной прибыли


# Что есть в игре

## Полноценный многопользовательский режим

Игроки подключаются в одну комнату и одновременно влияют на рынок

Можно:

- создавать публичные комнаты
- создавать приватные комнаты по коду
- играть с друзьями
- или заходить в случайные игры


## Система новостей и слухов

Каждый день компании получают:

- новости
- слухи
- сигналы роста
- сигналы падения

Но проблема в том, что слухи могут быть ложными

Игрок должен самостоятельно решить:

- верить информации
- или нет


## Инсайдерская информация

В игре можно покупать инсайды

Инсайд показывает точное направление будущего изменения цены компании:

- рост
- падение
- нейтральное движение

Но есть риск

Каждый инсайд повышает шанс проверки. Если игрока поймают — он потеряет часть капитала

Из-за этого приходится выбирать между:

- безопасной игрой
- и рискованной стратегией


## Рыночные пузыри

Если акции слишком сильно переоценены, появляется пузырь

Цена может расти очень быстро, но потом резко обвалиться

Это создаёт настоящую конкуренцию между игроками:

- кто успеет выйти раньше
- а кто останется с обесцененными активами

Технологическая компания

Особенности:

- новости появляются редко
- слухи появляются часто
- средняя вероятность правдивых слухов


## OilCore (OLC) 💧

Нефтяная компания

Особенности:

- самое большое количество новостей
- слухи чаще оказываются правдивыми
- более предсказуемое поведение


## HealthGen (HGN) 💊

Фармацевтическая компания

Особенности:

- большое количество слухов
- самая низкая вероятность правдивых слухов
- высокий уровень неопределённости

# Почему стоит попробовать

«Инсайдер» — это не просто игра про покупку и продажу акций

Это симуляция настоящего рынка, где игрок постоянно принимает решения в условиях неопределённости

В игре важно:
- рисковать или играть осторожно
- верить слухам или ждать подтверждений
- вовремя выходить из пузыря
- следить за действиями других игроков
- балансировать между прибылью и риском проверки

Даже одно неверное решение может привести к потере капитала, а удачный анализ рынка — к резкому росту состояния

Из-за случайных новостей, ложных слухов, пузырей и влияния игроков каждая партия проходит по-разному и создаёт ощущение живого нестабильного рынка

## Для разработчика

### Техническая архитектура

Проект реализован как клиент-серверное веб-приложение  
Frontend написан на HTML, CSS и JavaScript 
Backend реализован на Python с использованием Flask  
Для хранения данных используется SQLite

Пользователь взаимодействует с приложением через браузер
Frontend отвечает за отображение интерфейса, а backend — за игровую логику, комнаты, авторизацию и хранение состояния игры

Обмен данными между frontend и backend происходит через REST API в формате JSON

Для описания архитектуры используется модель C4:
- Context
- Container
- Component
- Code

## Таблица `users`

Хранит:
- id пользователя
- имя пользователя
- пароль


## Таблица `rooms`

Хранит:
- игровые комнаты
- статус игры
- текущий день
- код комнаты
- игровые данные

Статусы комнаты:
- `waiting`
- `playing`
- `finished`


## Таблица `room_players`

Хранит состояние игрока внутри комнаты:
- баланс
- риск
- акции
- действия
- события
- историю капитала


# REST API

| Метод | URL | Назначение |
|---|---|---|
| `POST` | `/api/register` | регистрация |
| `POST` | `/api/login` | вход |
| `GET` | `/api/me` | текущий пользователь |
| `GET` | `/api/rooms` | список комнат |
| `POST` | `/api/rooms` | создание комнаты |
| `POST` | `/api/rooms/<id>/join` | вход в комнату |
| `POST` | `/api/rooms/join-by-code` | вход по коду |
| `GET` | `/api/rooms/<id>/state` | состояние игры |
| `POST` | `/api/rooms/<id>/trade` | покупка или продажа |
| `POST` | `/api/rooms/<id>/insider` | покупка инсайда |
| `POST` | `/api/rooms/<id>/finish-day` | завершение дня |
| `POST` | `/api/rooms/<id>/events/ack` | подтверждение события |


# Основной игровой цикл

<img width="450" alt="IMAGE 2026-05-28 1:38:45 PM" src="https://github.com/user-attachments/assets/77ef6373-a220-4e19-a417-7c1bc0d189de" />


Игровой цикл:
1. пользователь входит в аккаунт
2. создаёт комнату
3. подключаются игроки
4. начинается игра
5. игроки совершают действия
6. сервер пересчитывает состояние
7. после 10 дней игра завершается
